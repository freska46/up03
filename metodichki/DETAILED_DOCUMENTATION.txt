==============================================================================
ЧАСТЬ 1: ОБЩАЯ АРХИТЕКТУРА
==============================================================================

Проект состоит из трех основных компонентов:

1. База данных PostgreSQL (pakdzal_games) - хранит данные
2. Web API (ASP.NET Core) - предоставляет данные по HTTP
3. WPF приложение - настольный клиент для отображения и редактирования данных

Связь между компонентами:
Пользователь → WPF приложение → HTTP запросы → Web API → PostgreSQL

Все три части работают независимо:
- PostgreSQL - отдельная программа (сервер БД)
- API - отдельное приложение (слушает порт 5222)
- WPF - отдельное приложение (запускается на компьютере пользователя)


==============================================================================
ЧАСТЬ 2: БАЗА ДАННЫХ (PostgreSQL + pgAdmin 4)
==============================================================================

Зачем нужна база данных:
- Хранит все данные приложения (пользователи, игры, сессии)
- Обеспечивает быстрый доступ к данным
- Поддерживает связи между таблицами

Структура базы данных:

Таблица USERS (Пользователи):
┌─────────────────┬──────────────────┬─────────────────────────────────────┐
│ Поле            │ Тип              │ Описание                            │
├─────────────────┼──────────────────┼─────────────────────────────────────┤
│ user_id         │ SERIAL PRIMARY KEY│ Автоинкремент, уникальный ID        │
│ name           │ VARCHAR(100) NOT NULL│ Имя пользователя (обязательно)      │
│ email          │ VARCHAR(100) UNIQUE│ Email (уникальный, обязательно)   │
│ phone          │ VARCHAR(20)       │ Телефон (необязательно)            │
│ city           │ VARCHAR(50) DEFAULT 'Курск'│ Город (по умолчанию Курск)    │
│ registration_date│ DATE DEFAULT CURRENT_DATE│ Дата регистрации               │
└─────────────────┴──────────────────┴─────────────────────────────────────┘

Почему такие типы данных:
- SERIAL - автоматически увеличивается на 1 при добавлении записи
- VARCHAR(n) - строка переменной длины до n символов (экономит место)
- NOT NULL - поле обязательно для заполнения
- UNIQUE - значение не может повторяться (один email на человека)
- DEFAULT - значение по умолчанию, если не указано

Таблица GAMES (Игры):
┌─────────────────┬──────────────────────┬────────────────────────────────┐
│ Поле            │ Тип                  │ Описание                       │
├─────────────────┼──────────────────────┼────────────────────────────────┤
│ game_id         │ SERIAL PRIMARY KEY    │ Уникальный ID игры             │
│ title          │ VARCHAR(100) NOT NULL│ Название игры                │
│ genre          │ VARCHAR(50) NOT NULL │ Жанр игры                    │
│ price_per_hour │ DECIMAL(10,2) NOT NULL│ Цена за час (до 999999.99)    │
│ available      │ BOOLEAN DEFAULT TRUE │ Доступна для аренды            │
└─────────────────┴──────────────────────┴────────────────────────────────┘

DECIMAL(10,2) - число с 10 знаками всего, 2 после запятой (например, 150.00)

Таблица SESSIONS (Сессии):
┌─────────────────┬───────────────────┬────────────────────────────────┐
│ Поле            │ Тип               │ Описание                       │
├─────────────────┼───────────────────┼────────────────────────────────┤
│ session_id      │ SERIAL PRIMARY KEY │ Уникальный ID сессии            │
│ user_id        │ INTEGER NOT NULL  │ Ссылка на пользователя (внешний ключ)│
│ game_id        │ INTEGER NOT NULL  │ Ссылка на игру (внешний ключ)   │
│ start_time     │ TIMESTAMP NOT NULL│ Время начала сессии            │
│ end_time      │ TIMESTAMP        │ Время окончания сессии         │
│ total_price   │ DECIMAL(10,2)   │ Общая стоимость                │
└─────────────────┴───────────────────┴────────────────────────────────┘

Внешние ключи (FOREIGN KEY):
- user_id REFERENCES Users(user_id) - связывает сессию с пользователем
- game_id REFERENCES Games(game_id) - связывает сессию с игрой

Связи между таблицами:
Users (1) ──────< Sessions (M) - один пользователь может иметь много сессий
Games (1) ──────< Sessions (M) - одна игра может использоваться во многих сессиях

ЧАСТЬ 3: WEB API (ASP.NET Core)
==============================================================================

Зачем нужно API:
- WPF не может напрямую работать с PostgreSQL
- API выступает посредником между WPF и базой данных
- API принимает запросы от WPF, обрабатывает, возвращает данные

Прото��ол HTTP:
- GET - получить данные
- POST - создать новую запись
- PUT - обновить существующую запись
- DELETE - удалить запись

Как работает HTTP запрос:
1. WPF отправляет запрос (например, GET /api/users)
2. API получает запрос
3. API обращается к базе данных через Entity Framework
4. Entity Framework формирует SQL запрос
5. PostgreSQL выполняет запрос
6. Результат возвращается обратно по цепочке

Точка входа Program.cs:
------------------------------------------------
using Microsoft.EntityFrameworkCore;           // Подключение EF Core
using PakDzal_Games_API.Data;                  // Наш контекст БД

var builder = WebApplication.CreateBuilder(args);

// Строка подключения - откуда брать настройки БД
var connectionString = builder.Configuration.GetConnectionString("DefaultConnection") 
    ?? "Host=localhost;Database=pakdzal_games;Username=postgres;Password=111";

builder.Services.AddDbContext<GameClubDbContext>(options =>
    options.UseNpgsql(connectionString));   // Настройка PostgreSQL провайдера

// Регистрация сервисов в DI контейнере
builder.Services.AddControllers();              // Добавляем контроллеры
builder.Services.AddEndpointsApiExplorer(); // Для документации API
builder.Services.AddSwaggerGen();          // Генератор Swagger

var app = builder.Build();

if (app.Environment.IsDevelopment()) // Только в режиме разработки
{
    app.UseSwagger();               // Включаем Swagger
    app.UseSwaggerUI();             // Интерфейс Swagger UI
}

app.MapControllers();                      // Сопоставляем маршруты
app.Run();

Почему используется DI (Dependency Injection):
- Модульность - легко заменить реализацию
- Тестируемость - можно подменить зависимости
- Переиспользование - один сервис в many местах

using System.ComponentModel.DataAnnotations;
using System.ComponentModel.DataAnnotations.Schema;

namespace PakDzal_Games_API.Models;

[Table("Users")]                          // Название таблицы в БД
public class User
{
    [Key]                                 // Это первичный ключ
    [DatabaseGenerated(DatabaseGeneratedOption.Identity)]
    public int UserId { get; set; }         // Свойство маппится на колонку

    [Required]                            // Обязательное поле (NOT NULL)
    [MaxLength(100)]                    // VARCHAR(100)
    public string Name { get; set; } = ""; // Значение по умолчанию

    [Required]
    [MaxLength(100)]
    public string Email { get; set; } = "";

    [MaxLength(20)]
    public string? Phone { get; set; }     // ? означает nullable (может быть null)

    [Required]
    [MaxLength(50)]
    public string City { get; set; } = "Курск";

    // virtual нужно для ленивой загрузки связанных данных
    public virtual ICollection<Session> Sessions { get; set; } = new List<Session>();
}

DbContext (GameClubDbContext.cs):
------------------------------------------------
using Microsoft.EntityFrameworkCore;
using PakDzal_Games_API.Models;

public class GameClubDbContext : DbContext
{
    // Конструктор принимает настройки через DI
    public GameClubDbContext(DbContextOptions<GameClubDbContext> options) : base(options) { }

    // Представляет таблицы в виде коллекций
    public DbSet<User> Users { get; set; } = null!;
    public DbSet<Game> Games { get; set; } = null!;
    public DbSet<Session> Sessions { get; set; } = null!;

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        base.OnModelCreating(modelBuilder);
        
        // Настройка сущности User
        modelBuilder.Entity<User>(entity =>
        {
            entity.HasKey(e => e.UserId);           // Первичный ключ
            entity.HasIndex(e => e.Email).IsUnique(); // Уникальный индекс для email
            entity.Property(e => e.City).HasDefaultValue("Курск"); // Default значение
        });

        // Настройка связей (Relationships)
        modelBuilder.Entity<Session>(entity =>
        {
            entity.HasOne(e => e.User)           // Сессия имеет одного пользователя
                  .WithMany(u => u.Sessions) // Пользователь имеет много сессий
                  .HasForeignKey(e => e.UserId) // Внешний ключ
                  .OnDelete(DeleteBehavior.Cascade); // Каскадное удаление
        });
    }
}

Контроллер UsersController.cs:
------------------------------------------------
using Microsoft.AspNetCore.Mvc;
using Microsoft.EntityFrameworkCore;
using PakDzal_Games_API.Data;
using PakDzal_Games_API.Models;

[ApiController]                            // Атрибут указывает что это API контроллер
[Route("api/[controller]")]              // Маршрут: api/users
public class UsersController : ControllerBase
{
    private readonly GameClubDbContext _context;

    // Внедрение через конструктор
    public UsersController(GameClubDbContext context)
    {
        _context = context;
    }

    // GET api/users - получить все�� пользователей
    [HttpGet]
    public async Task<ActionResult<IEnumerable<User>>> GetUsers(
        [FromQuery] string? search,           // Параметр из строки запроса
        [FromQuery] string? city)
    {
        var query = _context.Users.AsQueryable(); // IQueryable - для построения запроса

        if (!string.IsNullOrEmpty(search))
        {
            search = search.ToLower();         // Регистронезависимый поиск
            query = query.Where(u => u.Name.ToLower().Contains(search) || 
                                 u.Email.ToLower().Contains(search));
        }

        if (!string.IsNullOrEmpty(city))
            query = query.Where(u => u.City == city);

        return await query.ToListAsync(); // Выполнить запрос и вернуть
    }

    // GET api/users/1 - получить одного пользователя по ID
    [HttpGet("{id}")]
    public async Task<ActionResult<User>> GetUser(int id)
    {
        var user = await _context.Users.FindAsync(id);
        if (user == null) return NotFound(); // 404 если не найден
        return user;
    }

    // POST api/users - создать пользователя
    [HttpPost]
    public async Task<ActionResult<User>> CreateUser(User user)
    {
        _context.Users.Add(user);            // Добавить в контекст (еще не в БД!)
        await _context.SaveChangesAsync();    // Сохранить в базу данных
        return CreatedAtAction(nameof(GetUser), new { id = user.UserId }, user);
    }

    // PUT api/users/1 - обновить пользователя
    [HttpPut("{id}")]
    public async Task<IActionResult> UpdateUser(int id, User user)
    {
        if (id != user.UserId) return BadRequest(); // Проверка ID
        
        _context.Entry(user).State = EntityState.Modified; // Отметить как измененный
        try
        {
            await _context.SaveChangesAsync(); // Сохранить изменения
        }
        catch (DbUpdateConcurrencyException) // Конфликт обновления
        {
            if (!UserExists(id)) return NotFound();
            throw;
        }
        return NoContent(); // 204 успех без содержимого
    }

    // DELETE api/users/1 - удалить пользователя
    [HttpDelete("{id}")]
    public async Task<IActionResult> DeleteUser(int id)
    {
        var user = await _context.Users.FindAsync(id);
        if (user == null) return NotFound();

        _context.Users.Remove(user);    // Удалить из контекста
        await _context.SaveChangesAsync(); // Сохранить
        return NoContent();
    }
}

ApiService.cs (сервис для работы с API):
------------------------------------------------
using System.Net.Http;              // HTTP клиент
using System.Net.Http.Json;         // Работа с JSON
using PakDzal_Games_WPF.Models;

public class ApiService
{
    private readonly HttpClient _httpClient;
    private const string BaseUrl = "http://localhost:5222/api";

    public ApiService()
    {
        _httpClient = new HttpClient(); // Создаем HTTP клиент
    }

    // GET запрос
    public async Task<List<User>> GetUsersAsync(string? search = null, string? city = null)
    {
        var url = $"{BaseUrl}/users";
        
        // Добавляем параметры запроса
        if (!string.IsNullOrEmpty(search))
            url += $"?search={Uri.EscapeDataString(search)}";
        
        // Десериализация JSON в список User
        return await _httpClient.GetFromJsonAsync<List<User>>(url) ?? new List<User>();
    }

    // POST запрос - создание
    public async Task<User> CreateUserAsync(User user)
    {
        var response = await _httpClient.PostAsJsonAsync($"{BaseUrl}/users", user);
        response.EnsureSuccessStatusCode(); // Проверить успешность
        return (await response.Content.ReadFromJsonAsync<User>())!;
    }

    // PUT запрос - обновление
    public async Task UpdateUserAsync(User user)
    {
        var response = await _httpClient.PutAsJsonAsync($"{BaseUrl}/users/{user.UserId}", user);
        response.EnsureSuccessStatusCode();
    }

    // DELETE запрос - удаление
    public async Task DeleteUserAsync(int id)
    {
        var response = await _httpClient.DeleteAsync($"{BaseUrl}/users/{id}");
        response.EnsureSuccessStatusCode();
    }
}

using System.Windows;
using System.Windows.Controls;
using PakDzal_Games_WPF.Models;
using PakDzal_Games_WPF.Services;

public partial class MainWindow : Window
{
    private readonly ApiService _apiService = new();

    public MainWindow()
    {
        InitializeComponent();        // Инициализировать XAML
        Loaded += MainWindow_Loaded;   // Подписаться на событие загрузки
    }

    private async void MainWindow_Loaded(object sender, RoutedEventArgs e)
    {
        // Асинхронная загрузка данных при открытии окна

        {
            var users = await _apiService.GetUsersAsync();

        }
        catch (Exception ex)
        {
            MessageBox.Show($"Ошибка: {ex.Message}", "Ошибка");
        }
    }

    private async void btnAddUser_Click(object sender, RoutedEventArgs e)
    {
        var dialog = new UserDialog(); // Создать диалог
        dialog.Owner = this;           // Родительское окно
        if (dialog.ShowDialog() == true) // Показать модально
        {
            // Обновить данные
            var users = await _apiService.GetUsersAsync();
            dgUsers.ItemsSource = users;
        }
    }
}

Сценарий: Пользователь нажимает "Добавить" в WPF приложении

1. WPF: btnAddUser_Click вызывает new UserDialog()
2. UserDialog: Открывается окно с полями ввода
3. Пользователь заполняет данные и нажимает "Сохранить"
4. UserDialog: Создает объект User с введенными данными
5. UserDialog: Вызывает _apiService.CreateUserAsync(user)
6. ApiService: Создает POST запрос к /api/users
7. HTTP: Отправляет JSON на http://localhost:5222/api/users
8. API: Принимает запрос в UsersController.CreateUser()
9. API: Добавляет пользователя в контекст EF
10. API: Сохраняет в PostgreSQL (INSERT запрос)
11. API: Возвращает созданного пользователя
12. ApiService: Десериализует ответ в User
13. UserDialog: Закрывается с DialogResult = true
14. MainWindow: Обновляет DataGrid новыми данными


Сценарий: Пользователь нажимает "Отчет" в WPF приложении

1. WPF: btnUserReport_Click вызывает _apiService.GetUsersReportAsync()
2. ApiService: GET запрос к /api/users/report
3. API: Выполняет запрос к БД (SELECT COUNT(*) FROM users)
